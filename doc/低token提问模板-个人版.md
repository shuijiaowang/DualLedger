# 低 Token 提问模板（个人版）

## 使用方式
- 每次只处理一个任务，尽量开新会话。
- 先限定文件范围，再描述问题。
- 没必要的信息不要给，避免上下文膨胀。

## 通用模板（复制即用）
```md
【任务类型】修 Bug / 小功能 / 重构 / 文档
【目标】一句话说清最终结果
【只允许查看】<文件1>, <文件2>（最多 3-5 个）
【不要做】不要全项目扫描；不要改无关文件；不要跑全量构建
【已知信息】报错/现象/复现步骤（最多 5 行）
【输出要求】先给结论，再给修改点；代码改动尽量小
```

## 示例 1（后端）
```md
【任务类型】修 Bug
【目标】修复分类列表接口 500 错误
【只允许查看】service/api/category.go, service/service/category_service.go, service/dao/category.go
【不要做】不要扫描 web/；不要改路由和数据库结构
【已知信息】GET /api/category/list 返回 500；日志提示 nil pointer
【输出要求】直接改代码并说明根因，最后告诉我如何本地验证
```

## 示例 2（前端）
```md
【任务类型】小功能
【目标】首页新增本月收支概览卡片
【只允许查看】web/src/views/HomeView.vue, web/src/stores/meta.js, web/src/api/category.js
【不要做】不要改后端；不要动其他页面
【已知信息】已有接口可拿到分类和金额数据
【输出要求】只做最小改动，并给出手工验证步骤
```

## 低消耗习惯
- 一个问题一个会话，避免长历史累积。
- 不贴整篇长文档，只贴相关片段。
- 先问定位，再问修改，再问测试，分三轮更省。
